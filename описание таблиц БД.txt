1.Пользователь (User):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор пользователя.
Логин (строка): Имя пользователя для входа в систему.
Пароль (строка): Зашифрованный пароль пользователя.
Имя (строка): Имя пользователя.
Роль (строка): Роль пользователя в системе (например, администратор, сотрудник, обычный пользователь).

Внешний ключ: Связь с таблицей "Заказ" (Order) по полю "Пользователь_ID."

2.Категория товара (Category):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор категории товара.
Название (строка): Название категории товара.
Описание (текст): Описание категории товара.

3.Товар (Product):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор товара.
Название (строка): Название товара.
Описание (текст): Описание товара.
Цена (десятичное число): Цена товара.

Внешний ключ к таблице "Производитель" (Manufacturer) по полю "Производитель_ID."

4.Заказ (Order):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор заказа.
Дата заказа (дата и время): Дата и время размещения заказа.
Статус (строка): Статус заказа (например, "в обработке", "доставлен").

Внешний ключ к таблице "Пользователь" (User) по полю "Пользователь_ID."

5.Элемент заказа (OrderItem):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор элемента заказа.
Количество (целое число): Количество товаров, включенных в элемент заказа.

Внешний ключ к таблице "Товар" (Product) по полю "Товар_ID."
Внешний ключ к таблице "Заказ" (Order) по полю "Заказ_ID."

6.Журнал действий администраторов (AdminLog):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор записи в журнале.
Действие (строка): Описание действия, выполненного администратором.
Дата и время (дата и время): Дата и время выполнения действия.

Внешний ключ к таблице "Пользователь" (User) по полю "Пользователь_ID."

7.Производитель (Manufacturer):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор производителя.
Название (строка): Название производителя.
Страна производства (строка): Страна, в которой производится товар производителя.

8.Отзывы (Review):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор отзыва.
Текст отзыва (текст): Текст отзыва.
Рейтинг (целое число): Рейтинг товара от пользователя.

Внешний ключ к таблице "Товар" (Product) по полю "Товар_ID."
Внешний ключ к таблице "Пользователь" (User) по полю "Пользователь_ID."

9.Промокоды (PromoCode):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор промокода.
Код (строка): Уникальный код промокода.
Скидка (десятичное число): Размер скидки, предоставляемой промокодом (в процентах или фиксированной сумме).
Активен (логическое значение): Статус активации промокода.
Одноразовый (логическое значение): Флаг, указывающий, что промокод можно использовать только один раз.

10.Корзина (Cart):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор корзины.

Внешний ключ к таблице "Пользователь" (User) по полю "Пользователь_ID."

11.Сотрудники (Employee):
Поля:
ID (целое число, первичный ключ): Уникальный идентификатор сотрудника.
Имя (строка): Имя сотрудника.
Фамилия (строка): Фамилия сотрудника.
Должность (строка): Должность сотрудника.
Зарплата (десятичное число): Зарплата сотрудника.


СВЯЗИ:
Пользователь (User):
Связь с "Заказами" (Order): Один пользователь может разместить множество заказов (Один-ко-многим).
Связь с "Отзывами" (Review): Один пользователь может оставить множество отзывов (Один-ко-многим).
Связь с "Журналом действий администраторов" (AdminLog): Один пользователь может иметь множество записей в журнале (Один-ко-многим).
Связь с "Корзиной" (Cart): Один пользователь может иметь одну корзину (Один-к-одному).

Категория товара (Category):
Связь с "Товарами" (Product): Одна категория может содержать множество товаров (Один-ко-многим).

Товар (Product):
Связь с "Элементами заказа" (OrderItem): Один товар может быть частью множества элементов заказа (Один-ко-многим).
Связь с "Отзывами" (Review): Один товар может иметь множество отзывов (Один-ко-многим).

Заказ (Order):
Связь с "Элементами заказа" (OrderItem): Один заказ может содержать множество элементов заказа (Один-ко-многим).
Связь с "Промокодами" (PromoCode): Многие заказы могут использовать множество промокодов, и промокоды могут быть использованы в многих заказах (Многие-ко-многим).

Производитель (Manufacturer):
Связь с "Товарами" (Product): Один производитель может производить множество товаров, и каждый товар связан с одним производителем (Один-ко-многим).

Промокоды (PromoCode):
Связь с "Заказами" (Order): Многие заказы могут использовать множество промокодов, и промокоды могут быть использованы в многих заказах (Многие-ко-многим).

Корзина (Cart):
Связь с "Пользователем" (User): Один пользователь может иметь одну корзину (Один-к-одному).
Связь с "Товарами" (Product): Одна корзина может содержать множество товаров в себе. (Один-ко-многим)

Сотрудники (Employee):
Связь с "Заказами" (Order): Сотрудник может быть назначен для выполнения задач в заказах (Многие-ко-многим).
Связь с "Журналом действий администраторов" (AdminLog): Сотрудник работает только с одним журналом администратора (Один-ко-одному).
